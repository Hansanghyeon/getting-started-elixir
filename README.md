## 참고

엘릭서는 다른언어에 비해 값 사이의 비교를 비교적 너그럽게 허용한다. 다른 자료형 사이의 값 역시 비교가 가능하다. 같거나 비교 가능한 자료형이라면 일반적인 순서로 비교하고, 그렇지 않으면 다음과 같은 규칙으로 정렬한다.

- 숫자 < 아톰 < 레퍼런스 < 함수 < 포트 < 프로세스 ID < 튜플 < 맵 < 리스트 < 바이너리
  - nil이 아톰이므로 엘릭서에서는 숫자와 nul을 비교하면 항상 nil이 크다.
